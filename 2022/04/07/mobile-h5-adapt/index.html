1 概念

1.1 像素 & 分辨率

1.1.1 物理像素、物理分辨率

物理像素 设备显示屏上的渲染单位，其数量单位是px。

物理分辨率，设备显示屏的真实分辨率，用显示屏长宽上的物理像素点数表示，例如：iphone12物理分辨率 = 1170 * 2532。

1.1.2 逻辑像素、逻辑分辨率

逻辑像素 = css像素 ，其数量单位和设备相关，android中是dp，ios中是pt，前端开发过程中是px。

逻辑分辨率，设备系统设置的分辨率，当在设备中打开任何一个网页，都是以逻辑分辨率显示的。前端开发中使用的分辨率也是逻辑分辨率。

1.1.3 物理分辨率 && 逻辑分辨率 的关系

物理分辨率由设备硬件决定。设备出厂即决定，无法更改。

逻辑分辨率由设备软件决定，一般是操作系统，可以手动设置。

当 逻辑分辨率 < 物理分辨率 时，1逻辑像素 = n物理像素，一般设备系统设定的逻辑分辨率都是小于物理分辨率的，这样做一方面是为了适配，另一方面是为了让显示效果更细腻。

页面元素放大 不一定会导致 逻辑分辨率变小，但逻辑分辨率变小 一定会导致 页面元素放大。

1.2 视口（viewport）

1.2.1 设备与浏览器

设备和浏览器分为：

  • PC端：浏览器
  • 移动端：浏览器应用、app内嵌webview

和视口相关的参数有：

  • window.screen.width
  • window.innerWidth
  • document.documentElement.clientWidth

1.2.2 视口

布局视口（layout viewport）

能让网页上所有元素都正常显示出来的视口。由document.documentElement.clientWidth获取。

视觉视口（visual viewport）

通常布局视口中的内容不一定都能被用户看见，所以有了视觉视口去描述浏览器可视区域的大小。由window.innerWidth获取。

理想视口（ideal viewport）

用户不需要缩放或着滚动就能看见网页的所有内容。由window.screen.width获取。

1.3 设备概念

1.3.1 android系

dpi

dpi (dots per inch) = android屏幕密度，密度越高，屏幕显示能力越强。

dp 与 dip

dp = dip (device independent pixels) = 设备独立像素 = android的逻辑像素。

android中物理像素与逻辑像素的换算公式为：物理像素px = 逻辑像素dp * (dpi / 160)。

从公式中可以得到几点信息：

  • 在160dpi的android机上，1个逻辑像素等价于一个物理像素。
  • 通过公式可以算出android的逻辑分辨率，从而进行适配开发。

1.3.2 ios系

ppi

ppi (pixels per inch) = ios屏幕密度，密度越高，屏幕显示能力越强。

pt

pt （point） ios中用pt来描述逻辑分辨率。例如iphone12逻辑分辨率 = 390x844 pt，物理分辨率 = 1170x2532 px。

dpr

dpr (Device Pixel Ratio) 设备像素比 = window.devicePixelRatio

物理分辨率px / dpr = 逻辑分辨率pt，故可以用该公式计算ios的逻辑分辨率。

retina

视网膜屏幕，原意为：人视网膜无法分辨出屏幕像素点屏幕。代指dpr > 1的屏幕。

2 适配与高清方案

元素从设计稿到设备渲染展示，要经过两次尺寸的转化。

设计稿尺寸 —> 设备逻辑尺寸（逻辑分辨率） —> 设备物理尺寸（物理分别率）

在第一次尺寸转化中（即：交互稿尺寸转化成设备逻辑尺寸），面临一个问题：

同一套设计稿在不同逻辑分辨率的设备上如何呈现近乎一样的展示效果？

为了解决这个问题，自适应方案（也称适配方案）应运而生。

最常见的自适应方案是：rem

核心思想是：元素尺寸使用rem单位，用html根元素字体大小（px）的倍数表达元素的尺寸，这样，在不同逻辑分辨率的设备上，设置相应html根元素字体的大小（px），就能使得元素按比例展示适配该设备屏幕的尺寸。

相关库：

  • px2rem：用来在构建时将样式中px的大小数值换算成rem的比例数值，并缀上对应单位。所以只要配置好px2rem插件中根元素字体大小的值，在开发时就无需关注px换算rem的问题，直接按照设计稿尺寸一比一开发，打包构建时插件会自动换算。

  • flexible：用来设置html根元素字体的大小（px），在html中引入该script脚本，监听视口变化（resize、pageShow）并设置相应html根元素字体的大小（px）。

如何配置px2rem插件的根元素字体大小： flexible中html根元素字体的大小（px）= document.documentElement.clientWidth / 10 。所以px2rem配置的根元素字体大小值 = 设计稿宽度 / 10。这样配置开发时不用思考换算问题，设计稿原尺寸直出即可。

在第二次尺寸转化中（即：设备逻辑尺寸转化成设备物理尺寸）。也面临一些问题，需要用高清方案解决，比如：

0.5px border的实现

  • 渐变 background-image: linear-gradient + 媒体查询
  • 缩放 transform: scaleY + 媒体查询
  • 图片 + 媒体查询

图片的适配和优化

  • 多倍图 + 媒体查询：为不同dpr的设备提供最适合的图片尺寸
