1 概念

1.1 像素 & 分辨率

1.1.1 物理像素、物理分辨率

物理像素 是设备显示屏上的渲染单位，其数量单位是px。

物理分辨率，是设备显示屏的真实分辨率，用显示屏长宽上的物理像素点数表示，例如：iphone12物理分辨率 = 1170 * 2532。

1.1.2 逻辑像素、逻辑分辨率

逻辑像素 = css像素 ，其数量单位和设备相关，android中是dp，ios中是pt，前端开发过程中是px。

逻辑分辨率，是设备系统设置的分辨率，当在设备中打开任何一个网页，都是以逻辑分辨率显示的。前端开发中使用的分辨率也是逻辑分辨率。

1.1.3 物理分辨率 && 逻辑分辨率 的关系

物理分辨率由设备硬件决定。设备出厂即决定，无法更改。

逻辑分辨率由设备软件决定。设备出厂后，其系统设定了默认的逻辑分辩率，但该数值是可以被修改的（具体改法后续将详述）。

当 逻辑分辨率 = 物理分辨率 时，我们称之为”点对点显示”。

当 逻辑分辨率 < 物理分辨率 时，1逻辑像素 = n物理像素，一般设备系统设定的逻辑分辨率都是小于物理分辨率的，这样做一方面是为了适配，另一方面是为了让显示效果更细腻。

当 逻辑分辨率 > 物理分辨率 时，网页或软件呈现放大状态显示。

页面元素放大 不一定会导致 逻辑分辨率变小，但逻辑分辨率变小 一定会导致 页面元素放大。

2 设备与浏览器、视口（viewport）

2.1 设备与浏览器

设备和浏览器分为：

  • PC端
    • 浏览器
  • 移动端
    • 浏览器应用
    • app内嵌webview

要讨论网页展示就离不开以下几个参数：

  • window.screen.width
  • window.innerWidth
  • document.documentElement.clientWidth

这些参数与设备的关系如图所示：

在PC端，浏览器大小可以通过鼠标拖动改变的。故随着窗口大小的改变，window.innerWidth 和 document.documentElement.clientWidth的数值也会发生变化

在移动端，用户无法通过手势缩小放大浏览器或webview窗口大小，且移动端中无scrollbar。故这三个参数的数值相对稳定。

2.2 视口

布局视口（layout viewport）

能让网页上所有元素都正常显示出来的视口。由document.documentElement.clientWidth获取。

视觉视口（visual viewport）

通常布局视口中的内容不一定都能被用户看见，所以有了视觉视口去描述浏览器可视区域的大小。由window.innerWidth获取。

理想视口（ideal viewport）

用户不需要缩放或着scrollbar就能看见网页的所有内容。由window.screen.width获取。

3 设备概念

3.1 android系

dpi

dpi (dots per inch) = android屏幕密度，密度越高，屏幕显示能力越强。

dp 与 dip

dp = dip (device independent pixels) = 设备独立像素 = android的逻辑像素。

android中物理像素与逻辑像素的换算公式为：物理像素px = 逻辑像素dp * (dpi / 160)。

从公式中可以得到几点信息：

  • 在160dpi的android机上，1个逻辑像素等价于一个物理像素。
  • 通过公式可以算出android的逻辑分辨率，从而进行适配开发。

3.2 ios系

ppi

ppi (pixels per inch) = ios屏幕密度，密度越高，屏幕显示能力越强。

pt

pt （point） ios中用pt来描述逻辑分辨率。例如iphone12逻辑分辨率 = 390x844 pt，物理分辨率 = 1170x2532 px。

dpr

dpr (Device Pixel Ratio) 设备像素比。

dpr = window.devicePixelRatio = 1 / initial-scale

initical-scale 在meta标签中取

物理分辨率px / dpr = 逻辑分辨率pt，故可以用该公式计算ios的逻辑分辨率，活着直接通过window.screen.width查看逻辑分辨率。

retina

视网膜屏幕，原意为：人视网膜无法分辨出屏幕像素点屏幕。代指dpr > 1的屏幕。

4 适配方案

从设计稿到网页上线，要经历两次适配：

交互稿 —> (适配1) —> 逻辑分辨率 —> (适配2) —> 物理分辨率 —> 设备渲染

适配1：需要前端开发工程师自己适配
适配2：设备系统自动适配

故针对适配1，有两种适配方式：

  • 方式1：使用@meida媒体查询设备逻辑分辨率进行分配，这种方式的缺陷是有时无法1:1还原交互稿。
  • 方式2：通用适配方案：适配方案
