自制hexo的codepen插件

前言

博客中需要插入，codepen代码。

于是在hexo 官网 搜到了那么一个插件，幸运的是这个插件已经失效了。

虽然在npm官网上搜，依然能找到很多能使的插件：

开发hexo codepen插件。

如何在hexo中开发自定义插件

ref

有两种方式：

一种

首先博客根目录下（package.json同级目录下）添加script文件夹。

然后在script文件夹中添加你想开发的插件的js文件。

| - MyBolg
    | - package.json
    | - ...
    | - scripts // 自定义插件目录
        | - index.js

在js文件中，将插件函数注册到全局hexo变量的不同属性上即生效。

具体注册方法见文档

tips: 这里的hexo是通过npm i -g hexo全局安装的

另一种

首先在博客node_moudles目录下添加名为hexo-xxx的文件夹（命名必须以hexo-开头）

然后在该文件夹创建npm包相关的文件

| - hexo-xxx
    | - package.json
    | - package-lock.json
    | - index.js

最后根据hexo-xxx/package.json配置，在博客的package.json的dependencies字段填写相关信息。

这时自定义hexo插件便可以生效了。

hexo codepen插件原理

当不用插件时，使用codepen需要在hexo博客插入一段html。

这段html就是一个iframe标签，不同pen用标签src所带的不同hash值区分。

我们现在就是嫌手动把这段html插到md文档里不美观。

所以我们就写一个插件，这个插件用来在渲染时，接收从md文档中传入的hash参数，然后返回一串组装了该hash的html字符串模版给hexo渲染。

当然，插件接收的参数包括但不仅限于hash，也可以接收一些设定iframe样式的参数。

具体实现见源码，还是比较简单的。
